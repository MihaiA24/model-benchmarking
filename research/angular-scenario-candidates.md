# Angular/TypeScript scenario candidates

**Status:** Provisional research note for “Select the initial scenario portfolio”; this is not a frozen suite manifest.
**Scope:** Exactly four Public Suite candidates (one per accepted Workload Family) and four content-free Private Suite slot contracts. Repository heads were resolved to full commits on 2026-07-12. All measured checks must run from pre-provisioned, digest-pinned dependencies with network disabled.

## Provisional Public Suite selection

| Workload Family | Difficulty | Repository lineage | Candidate |
| --- | --- | --- | --- |
| Defect diagnosis and repair | **Challenging** | RealWorld Angular | Feed-route state identity regression |
| Bounded feature implementation | **Standard** | RealWorld Angular | Unsaved article-draft navigation protection |
| Test generation and hardening | **Standard** | Taiga UI | Numeric quantum transformer test hardening |
| Repository evolution | **Challenging** | NgRx Platform | Complete the Signal Events API rename migration |

This uses two RealWorld scenarios and one each from Taiga UI and NgRx; no lineage appears twice in one workload. These lineages must be excluded from the Private Suite.

### Policy provenance

- **Adopted:** the already accepted portfolio contract is applied unchanged here: one Angular/TypeScript scenario per Workload Family in each suite, behavior-level hidden acceptance plus regression checks, permissive immutable sources, and no measured network dependency.
- **Adapted:** observable-exposure review follows established benchmark contamination practice, but records public issues, patches, tests, and prior benchmark use rather than claiming knowledge of model training. The Taiga and NgRx concepts below extend first-party behavior instead of copying an upstream task verbatim.
- **Project-original:** the Public/Private split, 24-cell matrix, Standard/Challenging bands, lineage caps, exact scenario briefs/seeds/verifier projections, and pre-trial Private Suite commitment are local rules. No candidate is represented as a Terminal-Bench task. The RealWorld defect seed and draft-protection feature are project-original scenario designs.

## Rejected Public candidates

- **Legacy RealWorld missing-`@Input` repair and reading-time feature:** rejected from measured suites because the repair is token-level and build-only, while reading time is already assigned to the non-measured Legacy Calibration Suite and lacks sufficient repository integration.
- **RealWorld service `search(q)` feature:** rejected because the historical concept invents an API contract without first-party backend evidence and would depend on mutable external behavior unless a synthetic backend changed the nature of the task.
- **NgRx `rxMethod` injector-teardown defect:** rejected for this cell because [`rx-method.spec.ts`](https://github.com/ngrx/platform/blob/fa0780ee1a4ecd0ceead4566c11795041d5f12e4/modules/signals/rxjs-interop/spec/rx-method.spec.ts) already names and localizes the relevant lifetime cases around [`rx-method.ts`](https://github.com/ngrx/platform/blob/fa0780ee1a4ecd0ceead4566c11795041d5f12e4/modules/signals/rxjs-interop/src/rx-method.ts); a simple seeded conditional mutation would not credibly meet Challenging.
- **Angular Components `MatTableDataSource` or Sort Harness feature/test task:** rejected provisionally despite the [MIT license](https://github.com/angular/components/blob/c895595e6daaef11e8200603d25b9b9ed15102bb/LICENSE), strong [`table-data-source.ts`](https://github.com/angular/components/blob/c895595e6daaef11e8200603d25b9b9ed15102bb/src/material/table/table-data-source.ts) and [`table-data-source.spec.ts`](https://github.com/angular/components/blob/c895595e6daaef11e8200603d25b9b9ed15102bb/src/material/table/table-data-source.spec.ts) evidence, and pinned [`pnpm-lock.yaml`](https://github.com/angular/components/blob/c895595e6daaef11e8200603d25b9b9ed15102bb/pnpm-lock.yaml): the Bazel/browser toolchain and repository-scale dependency closure in [`package.json`](https://github.com/angular/components/blob/c895595e6daaef11e8200603d25b9b9ed15102bb/package.json) have not been shown to fit hermetic `standard-v1`, and the inspected surfaces are already heavily tested.
- **Taiga UI Angular-major monorepo upgrade:** rejected as open-ended under `standard-v1`; the snapshot spans many libraries and E2E projects, and no bounded compatibility slice with deterministic offline dependencies was established.
